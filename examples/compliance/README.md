# Compliance Example

Demonstrates how MIND's compiler toolchain produces auditable, regulation-ready AI artifacts. Every compiled binary carries a complete provenance chain from source code to deployed binary, enabling compliance with FDA, EU AI Act, and SLSA frameworks.

## Files

### `auditable_model.mind`

A two-layer MLP (8 -> 16 -> 1) annotated with provenance metadata. Shows:

- **Differentiable tensors** (`diff tensor<f32[dim1, dim2]>`) for autodiff-tracked parameters
- **Forward pass** with ReLU and sigmoid activations
- **Backward pass** via `backward(loss, param)` for gradient computation
- **Provenance annotations** documenting the full chain: source SHA256 -> IR SHA256 -> binary SHA256
- **Deterministic builds** -- identical source + compiler + flags produce bit-for-bit identical binaries

Build with provenance tracking:

```bash
mindc build auditable_model.mind --target cpu --provenance=full
```

This produces:
- `auditable_model.bin` -- the compiled binary
- `auditable_model.att` -- JSON provenance attestation

### `audit_report.mind`

A script demonstrating the full compliance artifact pipeline. Covers four stages:

1. **SBOM generation** -- Software Bill of Materials in SPDX 2.3 format
2. **SLSA provenance** -- in-toto attestation proving build integrity (Level 3)
3. **Build verification** -- hash comparison, signature validation, reproducibility check
4. **Compliance report** -- regulatory mapping for FDA 21 CFR Part 11 and EU AI Act

## Running the Audit Pipeline

### Step 1: Build the model

```bash
mindc build auditable_model.mind --target cpu --provenance=full
```

### Step 2: Generate SBOM

```bash
mind_audit sbom auditable_model.bin --format=spdx-json
# Output: auditable_model.sbom.json
```

### Step 3: Generate SLSA provenance attestation

```bash
mind_audit provenance auditable_model.bin --slsa-level=3
# Output: auditable_model.provenance.json
```

### Step 4: Verify the build

```bash
mind_audit verify auditable_model.bin \
  --attestation=auditable_model.provenance.json \
  --source=auditable_model.mind
# Exit code 0 = all checks passed
```

### Step 5: Generate compliance report

```bash
mind_audit compliance auditable_model.bin \
  --framework=fda,eu-ai-act \
  --output=compliance_report.pdf
```

### All-in-one

```bash
mind_audit full auditable_model.bin \
  --source=auditable_model.mind \
  --framework=fda,eu-ai-act \
  --slsa-level=3
```

## Regulatory Coverage

### FDA 21 CFR Part 11

- **11.10(a)** Validation -- reproducible builds verified via hash comparison
- **11.10(b)** Copies -- source, binary, and attestation archived together
- **11.10(c)** Record protection -- Ed25519 signed, tamper-evident attestation
- **11.10(d)** System access -- builds run in ephemeral, sandboxed environments
- **11.10(e)** Audit trail -- full provenance chain from source to binary
- **11.50** Signature -- Ed25519 digital signature on all attestations

### EU AI Act

- **Article 9** Risk management -- model architecture documented in source
- **Article 10** Data governance -- dataset SHA256 in training manifest
- **Article 11** Technical documentation -- SBOM + provenance + compliance report
- **Article 12** Record-keeping -- immutable attestation archive
- **Article 13** Transparency -- source code available for regulatory audit
- **Article 14** Human oversight -- decision threshold documented in model
- **Article 15** Accuracy and robustness -- test metrics in verification report
- **Article 17** Quality management -- full CI/CD pipeline with signed builds

### SLSA Level 3

- **Source integrity** -- signed commits in version control
- **Build integrity** -- hermetic, reproducible build via mindc
- **Provenance** -- non-falsifiable attestation generated by the builder
- **Isolation** -- build runs in an ephemeral, sandboxed environment
