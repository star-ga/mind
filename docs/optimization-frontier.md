# MIND Optimization Frontier

## 1. THESIS: The Pareto-Optimal Compiler Claim

MIND's wedge is not a single scalar performance claim. It is a position on a five-dimensional Pareto frontier where no other compiler is simultaneously dominant on all axes:

**Axis 1 — Correctness (via self-host + translation validation)**
MIND self-hosts: the pure-MIND bootstrap compiler reproduces its own mic@3 binary IR byte-for-byte against the reference Rust compiler. This is translation validation by construction — the self-host gate is a continuous correctness proof that the compiler correctly implements its own specification. No other production compiler (GCC, Clang, Cranelift) has a continuously-enforced self-reproduction test at the byte level.

**Axis 2 — Determinism (via canonical IR + confluent rewriting + fixed reduction orders)**
For pure-integer code (Q16.16, int8, int16), bit-identity across CPU/ARM/GPU is a mathematical certainty, not an empirical observation. The 2's-complement integer ALU specification leaves zero degrees of freedom; no ISA-conforming implementation can produce a different bit pattern. The signed evidence chain (trace_hash = SHA-256 of canonical mic@3 bytes) embeds a cryptographic commitment inside the artifact itself — not at the build-system level. No mainstream compiler does this.

**Axis 3 — Output Speed (via deterministic integer kernels)**
MIND's int8 kernels already beat OpenBLAS f32 by 2.02x single-core and cuBLAS int8 by 1.28x, both byte-exactly. The roadmap below shows how to extend this lead through provably-optimal extraction and verified peephole synthesis.

**Axis 4 — Compile Speed (via content-addressable IR cache + fast canonicalization)**
The 2.61µs scalar_math criterion baseline establishes a reference point. The roadmap targets near-zero recompilation cost for unchanged modules and a faster IR optimization pass through deterministic e-graph extraction.

**Axis 5 — Verifiable Evidence Chain (formally grounded optimality)**
Suciu, Wang, and Zhang (ICDT 2025, arXiv:2501.02413) establish a bijection between e-graphs and reachable deterministic tree automata. Under this formulation, the extracted representative is the canonical minimum-cost element of the smallest congruence generated by the declared ruleset. When MIND integrates equality saturation, the trace_hash stops meaning "hash of the IR we happened to produce" and starts meaning "hash of the canonical representative of the minimal algebraic congruence class under the declared ruleset" — a mathematical object, not an implementation artifact. This is a strictly stronger claim than any other compiler makes about its optimization decisions.

**What "theoretically unbeatable" means precisely:**
It means Pareto-optimality on the joint vector (correctness, determinism, output speed, compile speed, evidence chain) — not dominance on any single scalar. A compiler that is faster but non-deterministic does not dominate MIND. A compiler that is deterministic but lacks a signed evidence chain does not dominate MIND. The claim is that within the declared algebraic ruleset, MIND produces the unique canonical minimum-cost program, bit-identically, across all substrates, with a verifiable proof embedded in the artifact. This is achievable. It is also a claim no other compiler currently makes or can make without adopting MIND's architecture.

## 2. OPTIMIZATION ROADMAP

### (a) Ship-Now: Small Wins, Low Risk, Immediate Criterion-Gated

#### A1. Extend Canonical Commutative-Operand Ordering
- **Where it plugs in:** `src/opt/ir_canonical.rs`, the existing `reorder_commutative_ops` pass (already handles `Add`, `Mul` via `rhs < lhs` swap)
- **Expected win:** Eliminates a class of spurious trace_hash mismatch from construction-order differences; tightens the self-host byte-identity guarantee. Zero compile-speed overhead (single in-place pass). Zero output-speed impact.
- **Work needed:** Extend the commutative set to `BitAnd`, `BitOr`, `BitXor`, `Eq`, `Ne`, `Min`, `Max`, and any variadic reduction tree (canonicalize to left-to-right binary shape before mic@3 emission)
- **Determinism-preserving:** Yes — confluent normalization, same result regardless of rule application order
- **Effort:** Small (1–2 days)
- **Sources:** MIND `src/opt/ir_canonical.rs` (internal); Equality Saturation: A New Approach to Optimization, arXiv:1012.1802

#### A2. MAP Epilogue Canonical Sort Verification (Roundtrip Audit Gate)
- **Where it plugs in:** `src/ir/compact/v3/evidence.rs`, `emit.rs` — add a `roundtrip_audit()` function in the test suite
- **Expected win:** Catches encoder regressions before they silently corrupt the evidence chain. Zero production overhead (test-only). The MAP epilogue already documents the six canonicalization invariants (sorted keys, no duplicates, ULEB128 encoding, trace_hash = SHA-256 of pre-MAP bytes).
- **Work needed:** Implement `roundtrip_audit()` that re-parses emitted MAP and verifies: (a) all keys in lexicographic order, (b) no duplicate keys, (c) trace_hash field equals SHA-256 of bytes before MAP sentinel, (d) all values well-typed per the MAP schema
- **Determinism-preserving:** Yes — audit-only, no emission path change
- **Effort:** Small (1 day)
- **Sources:** MIND `src/ir/compact/v3/evidence.rs` (internal); RFC 8949: CBOR Determinism

#### A3. OmniBOR Artifact Dependency Graph in Evidence Chain
- **Where it plugs in:** `src/ir/compact/v3/evidence.rs` — additive field
- **Expected win:** Closes the supply-chain provenance gap. Makes MIND artifacts independently auditable by any OmniBOR tool. +32 bytes per artifact. No existing field or mic@3 body bytes change.
- **Work needed:** Add `evidence_chain.adr` field = SHA-256 Merkle root of (source file hashes, toolchain version string, all imported module trace_hashes). Compute during evidence chain emission.
- **Determinism-preserving:** Yes — additive, hash is deterministic over fixed inputs
- **Effort:** Small (1–2 days)
- **Sources:** OmniBOR: A System for Automatic, Verifiable Artifact Resolution across Software Supply Chains, arXiv:2402.08980

#### A4. Content-Addressable IR Cache (trace_hash-keyed clang bypass)
- **Where it plugs in:** Between `src/ir/compact/v3/emit.rs` (mic@3 emission) and `src/eval/mlir_build.rs` (clang invocation)
- **Expected win:** Near-zero recompilation cost for unchanged modules. The clang -O3 step currently dominates compile time for larger programs. Cache key = existing trace_hash (already computed). Cache hit = skip clang entirely, return cached `.o`. Enables cross-machine sharing on the fleet (S1/K1/R1/G1) when trace_hashes match.
- **Work needed:** Implement a disk-backed content-addressable store keyed by trace_hash. On miss: compile as normal, store output. On hit: return stored output, verify output hash matches before use. Cache invalidation: trace_hash changes when source changes (by construction).
- **Determinism-preserving:** Yes — cache key IS the canonical IR hash; cache correctness equals determinism
- **Effort:** Medium (3–5 days including cache eviction, verification, and fleet-sharing protocol)
- **Sources:** MIND `src/ir/compact/v3/evidence.rs` (trace_hash field); Ca-derivations, NixOS Wiki; Reproducible Builds: Increasing the Integrity of Software Supply Chains, arXiv:2104.06020

#### A5. Formal Semantic Foundation for Evidence Chain (Tree Automata Bijection)
- **Where it plugs in:** Documentation of `src/ir/evidence.rs` and the public evidence chain specification — no code change needed in this phase
- **Expected win:** Transforms the evidence chain's semantic content from "operationally reproducible" to "formally defined." The ICDT 2025 result (arXiv:2501.02413) gives MIND the vocabulary to state: the trace_hash commits to the canonical representative of the minimal congruence class under the declared ruleset. #### A6. Hydra-Style Bitwidth-Independent Peephole Rule Generalization (Offline)
- **Where it plugs in:** Offline tooling — produces a `.rules` file consumed by the peephole table loader in `src/opt/fold.rs`

Particularly valuable for MIND's Q16.16 arithmetic where the same algebraic identity applies across i32 and i64 accumulator variants.

- **Effort:** Small (2–3 days, Hydra runs as an offline tool)
- **Sources:** Hydra: Generalizing Peephole Optimizations with Program Synthesis, OOPSLA 2024, dl.acm.org/doi/10.1145/3649837

### (b) Medium: Meaningful Wins, Bounded Scope

#### B1. Deterministic E-Graph Extraction via egg with `--features deterministic`
- **Where it plugs in:** `src/opt/ir_canonical.rs` and `src/opt/fold.rs` — replace the sequential `fold -> reorder_commutative -> constant_fold` pass pipeline
- **Expected win:** 5–15% output-speed improvement on scalar kernels from algebraic simplifications that the sequential pass order misses (e.g., `x*2+x -> x*3`, double-transpose elimination). 3–8% fewer IR instructions before MLIR emission. Provably best-in-ruleset for any fixed ruleset, vs. current heuristic pass ordering.
- **Work needed:** Add `egg` as a dependency with the `deterministic` feature flag (this swaps all internal `HashMap`s for `IndexMap`s, making e-class iteration and extraction order fully reproducible across platforms and runs). Define an `ASTSize` cost function. Express MIND's algebraic identities (commutativity, `a+0=a`, `a*1=a`, double-transpose, reshape-reshape fusion) as egg rewrite rules. Replace the sequential pass pipeline with a single saturation call. Hash the extracted Instr sequence as the new basis for trace_hash.
- **Determinism-preserving:** Yes — the `deterministic` cargo feature is specifically designed for this; tie-breaking is by stable e-node ID insertion order
- **Effort:** Medium (1–2 weeks)
- **Sources:** egg: Fast and Extensible Equality Saturation, arXiv:2004.03082 (POPL 2021); egg GitHub, `Cargo.toml` `deterministic` feature

#### B2. SPORES-Style Relational Equality Saturation for Tensor Op Reordering
- **Where it plugs in:** MIND's high-level tensor IR (the `Dot`, `MatMul`, `Sum`, `Transpose`, `Reshape` instruction cluster) before MLIR lowering
- **Expected win:** 5–20% output-speed improvement on tensor computation graphs from operator-reordering rewrites — matching TENSAT's 16% improvement over TVM on neural network graphs. MIND already beats cuBLAS int8 by 1.28x; this widens the gap further. The ILP extraction step is small (10–50 tensor ops per module), solving in under 1ms.
- **Work needed:** Express tensor algebraic rules (Transpose(Transpose(x,p), inv(p)) = x; MatMul associativity and reordering; Sum(MatMul(A,B), axis=0) = MatMul(Sum(A,axis=0), B) when shapes permit) as egglog rules. Run to fixpoint. Extract via small ILP (deterministic: unique optimal, tie-breaks by lexicographic variable ordering). Verify rewrites against the existing 14-shape gate and byte-identity canaries.
- **Determinism-preserving:** Yes — ILP has a unique optimal; ties broken by lexicographic variable ordering (a fixed convention)
- **Effort:** Medium (2–3 weeks)
- **Sources:** SPORES: Sum-Product Optimization via Relational Equality Saturation, VLDB 2020, arXiv:2002.07951; TENSAT, MLSys 2021, arXiv:2101.01332

#### B3. Offline Souper-style SMT Synthesis: Verified Peephole Rule Database
- **Where it plugs in:** `src/opt/fold.rs` — a static, deterministic rule table applied as a fixed-priority pattern match before MLIR emission
- **Expected win:** 3–8% code-size reduction and corresponding throughput improvement on integer-heavy kernels. Souper achieved 4.4% on code that Clang -O3 already optimized; the gap is larger for a new IR like mic@3 with fewer hand-written peephole rules. No compile-time overhead from the solver (offline synthesis; runtime is pure pattern matching).
- **Work needed:** Run a Souper-like CEGIS synthesis loop (Z3 or Bitwuzla) offline against MIND's mic@3 integer-SSA operations. mic@3 is a fixed-width, explicitly typed, purely-integer SSA IR — the most favorable fragment for SMT-based synthesis. Verify each rule with Alive2. Export the verified rule table as Rust source. Apply as a deterministic pattern-match pass before MLIR emission.
- **Determinism-preserving:** Yes — the rule table is fixed and priority-ordered; same input always picks the same rule
- **Effort:** Medium (2–4 weeks for the offline synthesis pipeline; the runtime integration is small)
- **Sources:** Souper: A Synthesizing Superoptimizer, arXiv:1711.04422; First-Class Verification Dialects for MLIR, PLDI 2025, dl.acm.org/doi/10.1145/3729309; Minotaur: A SIMD-Oriented Synthesizing Superoptimizer, OOPSLA 2024, arXiv:2306.00229

#### B4. Treewidth-Bounded Optimal Extraction for Low-Complexity E-Graphs
- **Where it plugs in:** As the extraction step inside the egg-based e-graph pass (B1), replacing greedy extraction for small function bodies
- **Expected win:** 5–15% additional extraction quality improvement over greedy for expressions with shared subterms (greedy double-counts shared costs; optimal DP avoids this). Applicable to typical MIND scalar functions and loop bodies. Overhead: 2–5x slower extraction step than greedy, but extraction is a small fraction of total compile time. For e-graphs with treewidth ≤ 10 (most scalar function bodies after circuit simplification reduces treewidth by 40–80%), the DP is polynomial.
- **Work needed:** Implement the treewidth-bounded DP as an alternative `Extractor` trait implementation in egg (~300–500 lines of Rust, per the arXiv:2408.17042 paper). Compute treewidth via standard algorithms. Route small e-graphs (treewidth ≤ 10) through optimal DP; larger e-graphs (full Conv2d lowerings) remain on greedy.
- **Determinism-preserving:** Yes — DP over tree decomposition has a unique fixpoint; given the same e-graph structure and cost function, produces identical extraction on every substrate
- **Effort:** Medium (1–2 weeks)
- **Sources:** E-Graphs as Circuits, and Optimal Extraction via Treewidth, arXiv:2408.17042 (2024)

#### B8. Iterative Bi-Directional Optimization (IBO) for MLIR Pass Pipeline
- **Where it plugs in:** `src/opt/mod.rs` (MIND IR pipeline), as an opt-in compilation mode — not the default path
- **Expected win:** IBO (arXiv:2410.03120, 2024) outperforms exhaustive forward phase-order search on standard benchmarks by escaping local optima that any fixed pass sequence reaches. Estimated 2–10% throughput gain on compute kernels. Compile-speed cost: O(k) overhead where k is iteration count — bounded at 2–3 iterations per the paper.

- **Determinism-preserving:** Yes — each forward and reverse pass is individually deterministic; the procedure is reproducible given a fixed reverse-pass set and iteration bound
- **Effort:** Medium (2–3 weeks)
- **Sources:** Beyond the Phase Ordering Problem: Finding the Globally Optimal Code w.r.t. Optimization Phases, arXiv:2410.03120 (2024)

## 3. THE ONE HIGHEST-ROI CONCRETE WIN: Deterministic E-Graph Extraction (B1 expanded)

**Why this is the single highest-ROI first move:**
- It solves the phase-ordering problem at the MIND IR level with a proven production library (egg, POPL 2021, widely deployed)
- It has a dedicated `--features deterministic` cargo flag that was designed specifically to preserve bit-identical output
- It is a drop-in at `src/opt/`, building on the existing `ir_canonical` pass infrastructure
- It provides a formal foundation for the evidence chain's semantic content (arXiv:2501.02413) as a free side effect
- It unlocks the extraction-quality improvement from B4 (treewidth-bounded optimal extraction) as a follow-on with zero architectural change
- It does not require any change to mic@3 wire format, MLIR emission, the kernel tier, or the self-host infrastructure

**Implementation plan (determinism-gated, criterion-gated throughout):**

Phase 1 (Days 1–3): Scaffold and validate the determinism invariant

Add `egg = { version = "0.9", features = ["deterministic"] }` to `Cargo.toml`. Write a standalone `src/opt/egraph_canon.rs` that constructs an `EGraph` from a single MIND `IRModule` function body. Define an `ASTSize` cost function. Run a single saturation step with zero rewrite rules and verify the extracted instruction sequence is byte-identical to the input. This is the zero-regression baseline — the e-graph round-trips the IR without changing it. Run the full criterion suite (`cargo bench --bench simple_benchmarks --features "std-surface cross-module-imports" -- compile_small`) and confirm the baseline is within the standing ±10% gate (currently: `scalar_math 2.61µs`). Run the `fixed_point` self-host smoke test and confirm byte-identity is unchanged.

Phase 2 (Days 4–7): Algebraic ruleset, one rule at a time with gate verification after each

Add rewrite rules in order of expected frequency in MIND IR:

1. `a + 0 => a` and `0 + a => a` (additive identity)
2. `a * 1 => a` and `1 * a => a` (multiplicative identity)
3. `a * 0 => 0` and `0 * a => 0` (annihilator)
4. Commutative canonicalization: `a + b => b + a` when `id(b) < id(a)` — this replaces the existing `reorder_commutative_ops` pass
5. `Transpose(Transpose(x, p), inv(p)) => x` (double-transpose elimination)
6. `Reshape(Reshape(x, s1), s2) => Reshape(x, s2)` when shape is compatible (reshape fusion)

After each rule addition: run `cargo test`, run `fixed_point` self-host smoke, run criterion suite. Any criterion regression over 10% stops the addition; the rule is moved to a separate `--opt=eqsat` gate. Any self-host byte-identity regression is a hard blocker.

Phase 3 (Days 8–10): Wire into the production compilation path

Replace the sequential `fold -> reorder_commutative -> constant_fold` call sequence in `src/opt/mod.rs` with a single `egraph_canon::optimize(module)` call. The `trace_hash` computation remains over the final extracted Instr sequence — no change to the evidence chain format. Run the full test suite (893 tests + cross_substrate 7/7 + keystone 7/7 + self-host body_smoke) and confirm all pass. Measure output-speed gain on the existing GEMM benchmarks.

Phase 4 (Day 11–12): Documentation and gate formalization

Update `src/opt/ir_canonical.rs` header to document: "The e-graph extraction is deterministic under egg's `deterministic` feature: all internal `HashMap`s replaced by `IndexMap` (insertion-ordered); tie-breaking by stable e-node ID insertion order. Per Suciu, Wang, Zhang (ICDT 2025), the extracted expression is the canonical representative of the minimal congruence class generated by the declared ruleset." This directly upgrades the semantic content of trace_hash. Add the `egraph_canon` round-trip test to CI.

**Expected measurable outcomes after Phase 4:**

## 4. THEORETICAL LIMITS AND HOW MIND TURNS THEM INTO MOATS

### Limit 1: The Determinism Tax on Kernels

**The limit:** Enforcing bit-identical output across substrates precludes floating-point accumulation, which means MIND cannot use standard IEEE-754 float GEMM paths that are faster on some hardware configurations due to fused multiply-add (FMA) units with extended precision.

**The moat:** MIND already demonstrates that integer arithmetic is not a performance compromise — the int8 kernels beat cuBLAS int8 by 1.28x and OpenBLAS f32 by 2.02x, byte-exactly. The "determinism tax" is empirically negative: integer avoids the FMA precision management overhead that float compilers must handle. The VNNI path at 28.7 GMAC/s vs AVX2 12.2 GMAC/s demonstrates that the ISA increasingly rewards integer dot product natively. The moat is: for pure-integer code, bit-identity across any conforming ISA is a mathematical certainty (2's-complement integer ALU leaves zero degrees of freedom), while for float code it is impossible without explicitly serializing reduction order. Competitors cannot claim MIND's property without abandoning float accumulation entirely.

### Limit 2: The Undecidability of Global Superoptimization

**The limit:** Rice's theorem establishes that for any Turing-complete IR, the problem of finding the globally shortest equivalent program is undecidable. Practical superoptimizers escape by restricting to control-flow-free, acyclic dataflow graphs (Souper's fragment) or by bounding search depth. No finite rule set covers all possible optimizations, so equality saturation reaches ruleset-relative optimality, not global optimality.

**The moat:** MIND's wedge claim is not "globally optimal over all semantically equivalent programs" — that is undecidable and no compiler can claim it. The claim is: "the unique canonical minimum-cost program in the declared algebraic congruence class, with a signed hash of that canonical form embedded in the artifact." This is achievable (ICDT 2025 establishes the formal basis) and it is a strictly stronger claim than any other compiler makes. GCC, LLVM, and Cranelift do not state what optimization class their output belongs to or provide any proof of it. MIND's evidence chain is the proof. The limit becomes a moat because it forces the claim into a formally defined regime where MIND can win cleanly.

### Limit 3: The Ruleset-Relativity of E-Graph Optimality

**The limit:** E-graph extraction is optimal within the ruleset — but if the ruleset is incomplete for the theory, optimal extraction still misses rewrites not covered by any rule. Extending the ruleset requires adding rules; a complete ruleset for 64-bit integer bitvector theory is finite but large (~10^4 rules, per Souper's experience).

**The moat:** The offline Souper-style synthesis pipeline (B3) addresses this directly. mic@3 is the most favorable fragment for SMT-based synthesis: fixed-width, explicitly typed, purely-integer SSA IR with no undefined behavior and no aliasing. Z3 and Bitwuzla have complete decision procedures for bitvector theory — equivalence checks are exact, not approximate, within the acyclic integer-SSA fragment. The synthesis pipeline can run continuously (CI-triggered or nightly) as the ruleset is extended toward completeness. Each verified rule addition strictly narrows the gap between "optimal within the ruleset" and "globally optimal on the integer-SSA fragment." No other compiler has this architecture: their optimization rules are hand-written heuristics with no formal completeness claim.

### Limit 4: The PSPACE Hardness of Phase Ordering

**The limit:** The phase-ordering problem for pass sequences is PSPACE-hard. No polynomial-time algorithm finds the globally optimal pass sequence for an arbitrary program. IBO (arXiv:2410.03120) escapes this by applying reverse phases, but it is a practical heuristic that converges to a local optimum over the reachable equivalence class, not a polynomial-time exact solver.

**The moat:** MIND's response to this limit is architectural rather than algorithmic. By unifying optimization phases into a single equality-saturation step (the aegraph mid-end, C2), MIND eliminates the phase-ordering problem for the rules it holds: all rules are applied simultaneously until saturation, and a single deterministic extraction step produces the output. The phase-ordering problem is a constraint on pass-sequence compilers; it does not apply to equality-saturation compilers within their ruleset. For rules outside the e-graph (e.g., target-specific instruction selection, kernel scheduling), the IBO approach (B8) provides a practical escape bounded by a fixed iteration count. The moat: MIND's architecture inherently sidesteps the PSPACE lower bound for the optimization problems within its ruleset boundary.

### Limit 5: The Halting Problem for Convergence Verification

**The limit:** For arbitrary rule sets, equality saturation may not terminate — the e-graph can grow unboundedly if rules generate new terms faster than they unify existing ones. Production systems use iteration or node-count bounds, which means saturation is not always complete.

**The moat:** MIND's ruleset is restricted to algebraic identities over a fixed-width integer and tensor algebra. This is a decidable theory (Knuth-Bendix completion applies to finite algebraic structures; the bitvector theory has a finite Herbrand universe for any fixed bitwidth). The Myhill-Nerode criterion (arXiv:2501.02413) provides an algorithmic termination check: saturation is complete when the automaton has no further state splits, not just when no new facts are derived in one iteration. For MIND's ruleset, this is a finite and verifiable condition. The moat: MIND can formally verify that saturation is complete for its declared ruleset, a claim no heuristic pass-sequence compiler can make.
